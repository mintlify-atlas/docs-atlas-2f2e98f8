Before you install and run the toolkit, make sure the following are available on your machine.

## Node.js 22.14+

The project requires **Node.js 22.14 or later**. This is enforced in `package.json` via `"engines": { "node": ">=22.14.0" }`.

The repo ships with an `.nvmrc` file pinned to `22.14`. If you use [nvm](https://github.com/nvm-sh/nvm), you can install and activate the correct version automatically:

```bash
nvm install   # reads .nvmrc and installs 22.14
nvm use       # activates 22.14 for this session
```

To make it the default for all new terminal sessions:

```bash
nvm alias default 22.14
```

<Tabs>
  <Tab title="macOS / Linux (nvm)">
    Install nvm if you don't have it:

    ```bash
    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
    ```

    Then install Node:

    ```bash
    nvm install 22.14
    nvm use 22.14
    ```
  </Tab>
  <Tab title="macOS (Homebrew)">
    ```bash
    brew install node@22
    brew link node@22 --force
    ```
  </Tab>
  <Tab title="Windows">
    Download the Node.js 22 LTS installer from [nodejs.org](https://nodejs.org) or use [nvm-windows](https://github.com/coreybutler/nvm-windows):

    ```powershell
    nvm install 22.14.0
    nvm use 22.14.0
    ```
  </Tab>
</Tabs>

Verify your installation:

```bash
node --version
# v22.14.x or later
```

## Bun

The project uses **Bun** as its package manager and script runner. The `.bun-version` file pins the version to `1.3.7`.

<Tabs>
  <Tab title="macOS / Linux">
    ```bash
    curl -fsSL https://bun.sh/install | bash
    ```
  </Tab>
  <Tab title="Windows">
    ```powershell
    powershell -c "irm bun.sh/install.ps1 | iex"
    ```
  </Tab>
  <Tab title="npm">
    ```bash
    npm install -g bun
    ```
  </Tab>
</Tabs>

Verify your installation:

```bash
bun --version
# 1.3.7 or later
```

<Note>
  Bun is used for package installation (`bun install`), running workspace scripts (`bun run`), and linking the CLI globally. npm or yarn are not supported for this project.
</Note>

## Git

Git is required for the mod management features (`civ7 mod git`) and for cloning the repository itself.

<Tabs>
  <Tab title="macOS">
    Git ships with Xcode Command Line Tools:

    ```bash
    xcode-select --install
    ```

    Or install via Homebrew:

    ```bash
    brew install git
    ```
  </Tab>
  <Tab title="Linux">
    ```bash
    # Debian / Ubuntu
    sudo apt install git

    # Fedora
    sudo dnf install git
    ```
  </Tab>
  <Tab title="Windows">
    Download Git for Windows from [git-scm.com](https://git-scm.com/download/win) or install via winget:

    ```powershell
    winget install Git.Git
    ```
  </Tab>
</Tabs>

Verify your installation:

```bash
git --version
# git version 2.x.x
```

## Civilization VII

To pull official game data with `bun run refresh:data`, you need a local installation of **Civilization VII**. The CLI reads your install directory from `civ.config.jsonc`. See [Configuration](/installation/configuration) for how to set that path.

<Warning>
  If Civilization VII is not installed, you can still use the SDK to generate mod files. You just won't be able to run `civ7 data zip` or `civ7 data unzip` to extract game resources.
</Warning>

## OS support

The toolkit is developed and tested primarily on **macOS** and **Linux**. Windows is supported via Bun and native Node.js — the CLI binary resolves to `civ7.exe` when linked globally on Windows.

All path resolution in `civ.config.jsonc` supports `~` expansion and environment variables, so paths work correctly across platforms.
