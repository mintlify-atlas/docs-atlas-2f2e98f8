---
title: "Quickstart"
description: "Install the toolkit, pull game data, and build your first Civilization VII mod in about five minutes."
---

This guide walks you through the full flow from a fresh clone to a working mod file. You'll install dependencies, link the CLI, pull official game resources, and use the SDK to generate a custom unit.

<Note>
  Before you start, make sure you have **Node 22.14+**, **Bun**, and **Git** installed. See [Prerequisites](/installation/prerequisites) if you need help.
</Note>

<Steps>
  <Step title="Clone the repository">
    Clone the repo and move into it:

    ```bash
    git clone https://github.com/mateicanavra/civ7-modding-tools-fork.git
    cd civ7-modding-tools-fork
    ```
  </Step>

  <Step title="Install dependencies and link the CLI">
    Install all workspace dependencies and register the global `civ7` binary:

    ```bash
    bun install --frozen-lockfile
    bun run link:cli
    ```

    <Tip>
      After this step, open a new terminal session (or run `source ~/.zshrc` / `source ~/.bashrc`) so that the `civ7` command is available on your `PATH`.
    </Tip>
  </Step>

  <Step title="Pull game data">
    Use the CLI to extract official game resources from your local Civilization VII installation:

    ```bash
    bun run refresh:data
    ```

    This runs `civ7 data zip default` followed by `civ7 data unzip default`, placing the outputs in `.civ7/outputs/`. You only need to run this once.

    <Note>
      Make sure Civilization VII is installed on your machine. The CLI reads `installDir` from `civ.config.jsonc`. See [Configuration](/installation/configuration) for path details.
    </Note>
  </Step>

  <Step title="Create a mod with the SDK">
    Install the SDK in your mod workspace (or at the repo root for a quick test):

    ```bash
    bun add @mateicanavra/civ7-sdk
    ```

    Create a file called `my-mod.ts` with the following content. This example creates a custom scout unit that replaces the base-game scout, uses the army commander's visual, and adds localized text:

    ```typescript
    import { Mod, UnitBuilder, ACTION_GROUP_BUNDLE, UNIT_CLASS, UNIT } from '@mateicanavra/civ7-sdk';

    const mod = new Mod({
        id: 'my-first-mod',
        version: '1.0.0',
    });

    const unit = new UnitBuilder({
        actionGroupBundle: ACTION_GROUP_BUNDLE.AGE_ANTIQUITY,
        typeTags: [UNIT_CLASS.RECON, UNIT_CLASS.RECON_ABILITIES],
        unit: {
            unitType: 'UNIT_CUSTOM_SCOUT',
            baseMoves: 2,
            baseSightRange: 10,
        },
        unitCost: { cost: 20 },
        unitStat: { combat: 0 },
        unitReplace: { replacesUnitType: UNIT.SCOUT },
        visualRemap: { to: UNIT.ARMY_COMMANDER },
        localizations: [
            { name: 'Custom Scout', description: 'An enhanced scout unit' }
        ],
    });

    mod.add([unit]).build('./my-mod');
    ```
  </Step>

  <Step title="Build the mod">
    Run the script to generate the mod folder:

    ```bash
    bun run my-mod.ts
    ```

    This produces a `my-mod/` directory containing the `.modinfo` file and all XML files the game needs — units, type tags, localizations, icon definitions, and visual remaps.
  </Step>
</Steps>

## What just happened?

Calling `mod.add([unit]).build('./my-mod')` triggers the `UnitBuilder` to:

1. Generate a `TypeNode` and `UnitNode` for `UNIT_CUSTOM_SCOUT`
2. Emit `current.xml` with unit stats, costs, and type tags
3. Emit `visual-remap.xml` to give the unit the army commander's appearance
4. Emit `localization.xml` with the English name and description
5. Emit `icons.xml` with the icon definition
6. Write a `.modinfo` file wiring all XML files to their correct action group load phases

You never touched XML directly.
